## 第二部分：多节点冗余实验深度分析

### 2.1 3A+LTD=2%现象解释

**实验观察**：
```
配置: 3个A节点 + LTD
预期: 增加正确节点应提升性能
实测: 正确率暴跌至~2%
```

### 2.2 能量景观分析

**原因1：资源稀释**

```
每个A节点的信号纯度 = 70% / 3 ≈ 23%

能量景观变平坦：
- 单A: 深坑 (E ≈ -0.35)
- 3A: 浅坑 (E ≈ -0.10)
```

**原因2：节点间干涉**

```
3个A节点之间也存在竞争
每个节点试图将系统拉向自己的"正确"方向
但由于噪声，三个方向的向量和可能相互抵消
```

**原因3：LTD干扰**

```
LTD作用于C的同时，也可能影响A-A连接
负Hebbian学习不仅破坏C，也破坏了A之间的协同
```

### 2.3 数学分析

**多节点能量函数**：

$$E_{total} = \sum_{i=1}^{3} E(A_i) + E(B) + E(C)$$

当3个A节点各自有不同的"正确方向"时：

$$E_{total} \approx -\sum_{i=1}^{3} w_i \cos(\theta_i)$$

其中$\theta_i$是A_i与真实方向P的夹角。

**结论**：当多个"正确"信号不完全一致时，系统无法收敛到任何稳定点。

## 第三部分："物理极限"重新定义

### 3.1 原报告的问题

原报告声称"70%是物理极限"，存在以下问题：

1. **来源不清**：是实验设定，非物理定律
2. **缺乏证明**：未给出严格数学推导
3. **自相矛盾**：第13章又说"提升信源纯度可突破"

### 3.2 重新定义

**新定义**：

> **信源纯度约束下的性能边界（Source Purity Constrained Performance Boundary, SPCPB）**

$$\text{最大正确率} \leq \min(\text{信源纯度}, 1.0)$$

**数学表达**：

给定：
- $P(\text{correct}) = p$ (信源纯度)
- $\sigma^2$ (噪声方差)

则系统正确率 $C$ 满足：

$$C \leq p + O\left(\frac{\sigma^2}{N}\right)$$

**解释**：
- 首要约束：信源纯度p
- 次要因素：噪声和维度N
- 提升路径：提高p或增大N

### 3.3 与香农极限的类比

| 概念 | 香农通信极限 | 本研究极限 |
|------|-------------|-------------|
| 名称 | 信道容量 $C = B \log_2(1+SNR)$ | 信源纯度约束 $C \leq p$ |
| 本质 | 信息传输速率上限 | 共识正确率上限 |
| 突破方式 | 提升SNR | 提升信源纯度 |
| 数学基础 | 信息论 | 动力系统稳定性 |

**类比合理性**：
- 香农极限：给定信道条件下的最大可靠传输速率
- SPCPB：给定信源条件下的最大可靠共识正确率

## 第四部分：LTD机制收敛性分析

### 4.1 李雅普诺夫稳定性分析

**定义李雅普诺夫函数**：

$$V(x) = -x^T W x$$

**导数**：

$$\dot{V} = \frac{dV}{dt} = -\dot{x}^T W x - x^T \dot{W} x - x^T W \dot{x}$$

### 4.2 无LTD时的收敛性

**假设**：W是固定的

$$\dot{V} = -2x^T W^T W x = -2\|Wx\|^2 \leq 0$$

**结论**：系统收敛到局部极小值，但可能是错误的。

### 4.3 有LTD时的收敛性

**LTD规则**：

$$\dot{W}_{ij} = -\eta \cdot x_i x_j \cdot \mathbf{1}(\text{识别为错误})$$

**李雅普诺夫函数变化**：

$$\Delta V = V_{new} - V_{old} = -x^T \Delta W x$$

代入LTD规则：

$$\Delta V = \eta \cdot (x^T x)^2 > 0$$

**关键发现**：
- 无LTD：$\dot{V} \leq 0$ (收敛到错误吸引子)
- 有LTD：$\Delta V > 0$ (推高错误吸引子的能量)

**物理解释**：
- LTD将错误吸引子的能量从$E_C \approx -0.35$ 提升到$E_C \approx 0.00$
- 系统从收敛到错误，变为不收敛（或收敛到正确）

## 第五部分：机制干涉的数学本质

### 5.1 现象描述

| 单独LTD | 单独置信加权 | LTD + 置信加权 |
|---------|-------------|----------------|
| ~70% | ~68% | **~53%** |

### 5.2 干涉机制分析

**问题本质**：两个机制在**不同尺度**上操作

| 机制 | 操作尺度 | 效果 |
|------|---------|------|
| LTD | 能量尺度 (E) | 改变吸引子深度 |
| 置信加权 | 信号尺度 (x) | 改变节点贡献 |

### 5.3 干涉的数学表达

**单独LTD**：

$$x_{t+1} = \tanh(W^L x_t + \alpha \cdot x_t)$$

其中 $W^L$ 是LTD修改后的权重矩阵。

**置信加权**：

$$x_{t+1} = \tanh(W x_t + \beta \cdot C(x_t) \cdot x_t)$$

其中 $C(x_t) = |\text{mean}(x_t)|$ 是置信度。

**组合（干涉）**：

$$x_{t+1} = \tanh(W^L x_t + \alpha \cdot x_t + \beta \cdot C(x_t) \cdot x_t)$$

### 5.4 干涉原因

**尺度冲突**：

$$\alpha \cdot x_t \text{ vs } \beta \cdot C(x_t) \cdot x_t$$

- 第一项：恒定的节点间耦合
- 第二项：动态变化的信号放大

**反馈回路**：
1. LTD提升C的能量 → C的信号减弱
2. 置信加权检测到C的低置信 → 进一步减弱
3. 但A的信号也被同样处理
4. 系统陷入"过度抑制"状态

**相图分析**：

```
         错误吸引子
            /
           /   ← LTD移动
          /
    ------/-------  ← 干涉区域
        /
       /
    正确吸引子
```

当两个机制同时作用时，系统可能进入"灰色区域"，无法收敛到任何一个吸引子。

```python
# LTD核心实现
def apply_ltd(W, x, eta=0.5):
    """长期抑郁：削弱错误连接的权重"""
    delta_W = eta * np.outer(x, x)
    W_new = W - delta_W
    # 归一化防止发散
    W_new = W_new / np.linalg.norm(W_new) * 10
    return W_new
```
