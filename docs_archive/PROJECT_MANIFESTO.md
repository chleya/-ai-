# F:/system_stability/ 项目说明书

## 项目核心

**研究问题**: 在什么条件下，一个系统可以长期维持非平凡结构？

**研究范式**: 从"AI语言" → "系统语言"

---

## 核心词汇（仅允许）

| 禁止词 | 替代词 |
|--------|--------|
| 模型 | 状态空间 |
| 学习 | 演化规则 |
| 任务 | 约束 |
| loss | 时间 |
| reward | 稳定性 |
| 泛化 | 吸引子 |

---

## 系统定义

```
系统 S = (X, Φ, C, T)

X: 状态空间 (state space)
Φ: 演化规则 (evolution rules)
C: 约束 (constraints)
T: 时间 (time)
```

**唯一问题**: 在给定约束下，演化是否会进入某种稳定相？

---

## 吸引子类型

| 类型 | 系统命运 |
|------|---------|
| 无吸引子 | 状态漂移 → 信息熵爆炸 → 死亡 |
| 只有固定点 | 系统僵死 |
| 只有混沌 | 系统不可控 |
| **非平凡吸引子** | 有结构、有区分度、可恢复 ← 目标 |

---

## 工作方法论

1. **固定一切，只放开一个自由度**
2. **观察长期行为**（不是短期表现）
3. **用相图、轨道、稳定性代替指标**
4. **提前接受失败是主结果**

---

## 第一步行动

```
1. 写下一个最小系统定义
2. 跑它
3. 看它会不会烂掉

最小系统:
- 64维状态
- 简单演化
- 一个归一化约束
```
