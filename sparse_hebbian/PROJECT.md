# Sparse Hebbian: Reducing Interference Through Structure

**基于**: System Stability 项目 (FINAL_REPORT.md)  
**日期**: 2026-02-07  
**起点**: 71% 模式干扰率

---

## 研究背景

### 核心问题

在 System Stability 研究中，我们发现Hebbian学习存在物理极限：

| 发现 | 值 |
|------|------|
| 模式干扰率 | 71% |
| 干扰机制 | 相似模式共享连接，产生"粘连" |

### 生物学启示

生物大脑皮层是极度稀疏的：
- 神经元连接密度：~10-20%
- 每个神经元只与少数其他神经元相连
- 稀疏性可能是减少干扰的关键

### 假设

> 砍掉80%的弱连接可以大幅降低干扰率，同时保持模式补全能力。

---

## 研究问题

### 主问题

稀疏Hebbian能否将71%的干扰率降低到可接受水平？

### 子问题

1. **稀疏度 vs 性能**: 砍掉多少连接是最优的？
2. **补全能力**: 稀疏化是否损害10%→100%的补全能力？
3. **抗噪能力**: 稀疏化是否影响噪声鲁棒性？
4. **容量变化**: 稀疏化如何影响存储容量？

---

## 实验设计

### 基线（来自System Stability）

| 指标 | 值 |
|------|------|
| 干扰率 | 71% |
| 补全率 | 100% (10%输入) |
| 抗噪 | 80% (σ=10) |
| 容量 | ~0.14N |

### 实验变量

| 稀疏度 | 含义 |
|--------|------|
| 0% | 全连接（基线） |
| 50% | 砍掉一半连接 |
| 80% | 砍掉80%连接 |
| 90% | 只保留10%连接 |
| 95% | 只保留5%连接 |

### 核心实验

#### 实验1: 干扰率测试

```
1. 创建2个正交模式 P1, P2 (N=50)
2. 应用不同稀疏度
3. 训练W在两种模式下
4. 测试恢复率
5. 记录干扰率
```

#### 实验2: 模式补全测试

```
1. 学习一个模式
2. 应用稀疏化
3. 测试10%→100%补全率
```

#### 实验3: 抗噪测试

```
1. 学习一个模式
2. 应用稀疏化
3. 测试不同噪声下的恢复率
```

### 悲观假设

| 稀疏度 | 干扰率 | 补全率 | 抗噪 |
|--------|--------|--------|------|
| 0% | 71% | 100% | 80% |
| 80% | 65% | 85% | 60% |
| 90% | 60% | 70% | 50% |

### 理论分析

**为什么稀疏可能减少干扰？**

1. **连接隔离**: 每个模式使用独立的连接子集
2. ** basins分离**: 稀疏W的 basins 更"尖锐"
3. **正交性增加**: 稀疏化增加了有效正交性

**为什么稀疏可能降低性能？**

1. **信息丢失**: 砍掉太多连接会丢失信息
2. ** basins变浅**: 稀疏W的 basins 更浅
3. **泛化能力下降**: 抗噪能力可能受损

## 关键假设检验

### H1: 稀疏化降低干扰

$$H_0: \text{稀疏干扰率} = \text{全连接干扰率}$$
$$H_1: \text{稀疏干扰率} < \text{全连接干扰率}$$

### H2: 稀疏化保持补全

$$H_0: \text{稀疏补全率} < \text{全连接补全率}$$
$$H_1: \text{稀疏补全率} \approx \text{全连接补全率}$$

### H3: 存在最优稀疏度

$$\exists s^* \in [0, 0.95] \text{ s.t. } F(s^*) > F(s) \forall s$$

其中F是综合性能指标。
